这篇文章专门用来检查当前博客对 Markdown 常见元素的样式支持是否稳定。

## 标题

你可以通过不同级别的标题来组织内容层级。

# H1

## H2

### H3

#### H4

## 段落

这是一段普通正文。个人博客通常最重要的不是首页有多花，而是正文能不能让人稳定读下去。

段落之间保持清晰的节奏，往往比堆很多视觉元素更重要。

## 引用与列表

> 像素风最好让结构承担个性，让正文保留克制。

1. 先处理结构，再处理风格
2. 先保证手机能看，再增强桌面端
3. 用少量变量统一颜色、边框和阴影

## 表格

| 模块 | 当前状态 | 后续可调整 |
| --- | --- | --- |
| 首页 | 已完成 | 字体、插画、装饰图形 |
| 文章列表 | 已完成 | 卡片层级和筛选方式 |
| 文章页 | 已完成 | 目录、脚注、目录锚点 |

## 代码块

```ts
const site = {
  framework: 'Astro',
  content: 'Markdown',
  style: 'Pixel Minimal',
  mobileFirst: true,
};
```

## 行内样式

你也可以在正文中使用 `inline code`、**加粗** 和 _强调_ 来增强表达。
