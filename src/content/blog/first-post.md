这篇文章是当前博客的第一篇占位内容，用来确认 Markdown 渲染、文章列表和详情页都已经正常工作。

## 为什么先做骨架

如果一开始就把时间花在视觉风格上，通常会很快陷入细节里。更合理的做法是先把这些部分立住：

1. 首页的信息层级
2. 文章列表的可扫描性
3. 正文页的阅读宽度和段落节奏
4. 移动端导航和按钮的可点击区域

## 这一版已经包含什么

- Astro 的静态站点结构
- Markdown 文章内容流
- 一个可继续微调的像素感样式基线
- 对手机屏幕优先的布局处理
