当前博客主要按 Markdown 使用，但项目里仍然保留了 MDX 支持。

## 为什么留着 MDX

如果后面你想在文章里嵌入组件，例如：

- 提示框
- 项目卡片
- 时间线
- 可折叠的补充内容

那 MDX 会比纯 Markdown 更灵活。

## 一个简单示例

import HeaderLink from '../../components/HeaderLink.astro';

<HeaderLink href="/blog">
	Back To Blog
</HeaderLink>

## 什么时候不用 MDX

如果你只是写纯文章，Markdown 就够了。只有在文章需要嵌入组件时，再考虑用 MDX，不要把它变成默认写作路径。
